для того, чтобы заставить работать это приложение из-под ОС Windows:

1 скачать python3.6 и установить его
2 через командрную строку зайти в репозиторий в папку venv_flask_b/Scripts/
3 ввести activate 
  после чего должно активироваться виртуальное окружение (в командной строке справа будет: (venv_flask_b) - такая скобка)
4 поднимаемся в каталог, где лежит файл blog.py
5 нужно создать переменную виртуального окружения для запуска приложения. Вводим:
set FLASK_APP=blog.py
6 теперь просто вводим команду flask run
7 сервер запущен, должно появиться что-то в роде:
* Server Flask app "blog"
* Running on http://127.0.0.1:5000

для запуска сервера на linux:

нужно создать виртуальное окружение (для windows оно уже было создано и помещено в репозиторий), для linux это придется сделать самостоятельно:
0 устанавливаем python 3.5 или выше
1 устанавливаем инструмент для создания виртуального окружения (в python3.6 он уже есть из коробки). Вводим в терминале:
pip3 install virtualenv
2 теперь нужно выбрать папку, куда будет помещена папка с виртуальным окружением. Заходим в нее из-под терминала и вводим:
python3 -m venv <название_виртуального_окружения>
название окружения лучше вводить строчными английскими буквами без пробела (но можно с знаками подчеркивания). 
Теперь в этой папке будет создана папка с виртуальным окружением
3 заходим из-под терминала в эту папку, далее в папку bin и далее вводим следующее:
source activate
теперь виртуальное окружение активировано и его название должно появиться в круглых скобках слева в терминале
4 теперь нужно установить следующие пакеты для python, - вводим:
pip3 install flask
так мы установили фреймворк flask
теперь нужно установить расширения для него:
pip3 install flask-wtf flask-sqlalchemy flask-moment flask-bootstrap flask-migrate
5 теперь создадим переменную окружения для запуска приложения. Переходим в каталог с файлом blog.py и вводим:
export FLASK_APP=blog.py
6 теперь вводим команду flask run
7 сервер запущен, должно появиться что-то в роде:
* Server Flask app "blog"
* Running on http://127.0.0.1:5000

===================
также можно установить значение среды окружения для запуска сервера в режиме дебага:

set FLASK_DEBUG=1
или
export FLASK_DEBUG=1
в зависимости от ОС

подробнее: flask run --help
в мультипоточном режиме на порте 12794 для всех:
flask run --port 12794 --host 0.0.0.0 --with-threads
мультипоточный режим работает только при выключенном дебаге
--------------------
чтобы запустить приложение с доп. настройками
в файле __init__.py внизу пропиши app.run() с желаемыми параметрами запуска. Что-то в роде:

app.run(host='0.0.0.0', port=8080, debug=True)

v6.2 +
чтобы сообщения с логгера приходили на эл. почту нужно установить переменные окружения:
MAIL_SERVER
MAIL_PORT
MAIL_USERNAME
MAIL_PASSWORD
TO_MAILS
во всех случаях пишем без кавычек
В TO_MAILS чтобы указать несколько email'ов, их нужно записать через запятую (и тоже без кавычек и БЕЗ ПРОБЕЛОВ)
